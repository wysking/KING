[1][2]
。本文依据工厂提供的合炉规
定（表 1）
[10][11]
和订单数据，分别针对固溶工艺、淬火工艺及无工艺区分三种情景，
建立了排产优化模型。
首先，通过数据清洗与特征工程，提取合金大类、厚度组、合炉厚度差上限、必须

[10]
。其次，按温度和合炉规则将物料分组形成初始炉批，同一

[4][7]
，确保相邻板材厚度差≤30%、宽

炉批按温度升序衔接，检查并调整边界处的温差、厚度差和宽度差，必要时合并或
反转炉批顺序以满足约束
[1][8]
。最后，根据工艺类型为每个炉批生成对应的固溶或淬
火工艺参数说明（固溶工艺采用炉批内最厚板材参数；淬火工艺中，若炉批内最大
厚度≥50mm 则采用最厚参数，否则采用中间厚度参数）
[10]
。
实验结果表明，算法能在毫秒级内完成数百条订单的排产，生成的炉批数合理，相
邻物料约束满足率达 100%，且工艺参数完全符合表 1 规定

1.问题重述
1.1 背景
辊底炉（辊底式炉）是一种连续式工业加热炉，炉底由电动旋转的耐热辊棒构成，
工件直接放在辊面上，被辊子带动匀速通过炉膛，完成加热或热处理
[7]
。排产的核
心要求是：按材质、厚度、工艺和温度分组形成“炉批”，不同炉批切换时需满足
厚度、宽度和温度的渐变约束，避免辊速剧烈调整导致加热不均
[1][8]
。
1.2 约束条件
根据题目描述，辊底炉排产需满足以下约束：
（1）同温度物料内部接序要求
相邻宽度：宽→窄，宽度差≤800mm；窄→宽，宽度差≤500mm。
相邻厚度：厚度差≤30%，计算公式为 (厚 − 薄)/厚 × 100%。
在同温度范围内，满足上述要求的物料可连续生产；不满足的需分成不同小组
[7]
。
（2）不同温度物料小组间接序要求
相邻物料温度：低温→高温，温度差≤50℃；高温→低温，温度差≤20℃
[7]
。
同时满足上述宽度差和厚度差要求。
在满足温度接序的前提下，优先保证宽度相同或接近的物料相邻。
（3）合炉规定（表 1-1）
[10][11]
合金分为硬铝系（2024、2A12、7050、7075、7A04、7A05）和铝镁硅系（6061、6082、
6063）。

2.模型假设与符号说明
2.1 模型假设
为简化问题并聚焦核心约束，本文做如下合理假设
[2][4]
：
1.炉批内物料可任意排序：不考虑炉批内物料的生产顺序对设备磨损或能耗的差异，
仅需满足相邻约束即可
[7]
。
2.温度切换无额外代价：不同温度炉批之间的切换时间、能耗成本不计入优化目标，
仅需满足温差约束。
3.订单均为“待生产”状态：所有订单在排产开始时均未生产，不考虑正在进行的
批次或设备占用。
4.交货日期准确且紧急标记优先：以“紧急标记”为最高优先级，“交货日期”为
次优先级，拖期标记仅作为参考
[4]
。
5.同一炉批内工艺参数一致：固溶工艺取炉批最厚板材参数；淬火工艺按规则取中
间厚度或最厚厚度参数
[10]
。
6.忽略炉长限制：假设辊底炉无限长，不考虑单个炉批总长度是否超出炉膛物理长
度（实际中可通过多炉次解决，本文暂不计入）
[8]
。

4.问题 1：固溶工艺排产模型与求解
4.1 问题描述
在固溶工艺下，需将所有订单板材按温度、合金、厚度分组，合炉形成炉批，每个
炉批内相邻板材满足厚度差≤30%、宽度差方向性要求，不同炉批之间还需满足温
差（升温≤50℃、降温≤20℃）。最终为每个炉批生成固溶工艺描述：采用该炉批
内最厚板材对应的工艺参数
[10]
。

4.2 算法设计与迭代改进
本文针对固溶工艺设计了三代排产算法，逐步完善合炉规则、排序策略和约束检查。
整体流程如图 4-1 所示。
图 4-1
4.2.1 第一代算法（初始版本）
第一代算法按温度分组，在每个温度内按合金大类与厚度组_合炉进一步分组。对于
允许合炉的小组，按厚度排序后贪心拆分厚度极差≤允许上限的子组；对于不允许
合炉的小组，每块板材单独成组。然后调用 DeepSeek API 对每个子组进行智能排序，


===== PAGE 11 =====
优先安排紧急、早交期板材。最后按温度升序、同温内按优先级降序排列所有炉批，
并检查相邻边界，若违规则合并两个小组重新排序。
存在问题：
1.未严格处理“必须单独一炉”的物料，导致<20mm 硬铝系或<30mm 铝镁硅系板材可
2.能被错误合并
[10]
。
3.不同温度组之间的边界处理简单，仅靠合并可能引入温度混杂。
4.部分相邻板材仍会出现厚度差>30%或宽度差违规，自检通过率仅约 95%。
4.2.2 第二代算法（修正版本）
针对第一代的问题，第二代算法进行了以下改进：
强化必须单独一炉的处理：在分组阶段，将必须单独一炉 == 1 的物料单独成批，
绝不与任何其他物料合并。
引入厚度极差严格检查：对每个允许合炉的组，按厚度升序后，只有新板材与组内
最小厚度的差值≤允许上限时才加入当前炉批，否则关闭当前批并开启新批。
增加全局相邻约束强制拆分：遍历所有相邻板材，若违反厚度差或宽度差约束，则
在违规位置强制切分炉批号。
不同温度组边界处理：在合并时，对相邻小组检查温度差，若不符合升温≤50℃或
降温≤20℃，则尝试交换顺序或发出警告
[7]
。
遗留问题：
API 排序偶尔返回不满足约束的顺序，降级为本地排序后效果一般。
不同温度组的衔接仍存在少量温差违规（例如 335℃→290℃温差 45℃，超过 20℃
限制）。
部分合炉炉批内的厚度极差实际仍超出允许值（如 20-50mm 组允许 5mm，但实际极
差达到 8mm）。

5.问题 2：淬火工艺排产模型与求解
5.1 问题描述
淬火工艺与固溶工艺的主要区别在于工艺参数选取规则
[10]
：
若炉批内所有板材厚度<50mm，则采用中间厚度对应的淬火参数。
若炉批内存在板材厚度≥50mm，则采用最厚厚度对应的淬火参数。
5.2 算法迭代改进
5.2.1 第一代算法（API 优化版）
第一代淬火算法直接复用问题一第一代的分组和 API 排序逻辑，并在生成炉批后添
加淬火工艺描述。存在以下问题：
1.淬火参数选取错误：未区分全组是否都<50mm，导致部分炉批错误地使用了中位数。
2.炉批 16（特定批次）内部厚度范围过大（15mm–160mm），厚度极差远超允许上
限，违反合炉规定。
3.边界温度差检查不严格，出现 335℃→290℃降温 45℃（应≤20℃）的违规。
5.2.2 第二代算法（修正版）
针对上述问题，第二代算法进行了以下修正
[8][10]
：
1.拆分违规炉批 16：按厚度区间（<20、20-50、50-100、150-260）拆分为多个独
立炉批，确保每个炉批内厚度极差≤允许上限。
2.淬火参数逻辑修正：
if max(thicknesses) < 50:
ref_thick = median(thicknesses) # 全部<50mm，取中间厚度
else:
ref_thick = max(thicknesses) # 存在≥50mm，取最厚厚度
3.全局顺序重组：按温度升序、同温内按厚度中位数降序排列炉批，使厚板在前、
薄板在后。
4.边界温度差检查：对相邻炉批计算温差，若升温>50℃或降温>20℃，则强制拆分
或调整顺序；若仍无法满足，记录警告（实际数据中未出现）。

6.问题 3：无工艺区分排产模型与求解
6.1 问题描述
问题三不考虑固溶或淬火工艺，仅需满足温度、厚度、宽度三项约束，不生成工艺

[7][10]
。
6.2 算法迭代过程
6.2.1 第一代至第五代（逐步改进）
第一代：全局按紧急、交期、厚度降序排序，按温度分组，组内依次分配炉批号，
只要相邻满足厚度非递增且厚度差≤30%、宽度差合规，就归入同一炉批。问题：
未利用表 1- 1 的厚度分组和允许厚度差，导致炉批内厚度极差可能超标；未处理“必
须单独一炉”；不同温度组间未检查温差。
第二代：引入厚度区间分组（<30、30-100、100-300），组内按厚度降序排序，初
步控制厚度范围。
第三代：增加代表厚度概念，按厚度中值排序温度组，优化组间边界衔接，温度组
间温差违规减少 80%。
第四代：动态调整每个温度组内炉批的首尾物料，以匹配前后组边界，边界合规率
提升至 95%。
第五代：加入炉长限制（20000mm），避免单个炉批总长度超限，更贴近实际生产
[8]
。
6.2.2 第六代（严格合炉）与第七代（最终版）
第六代算法实现了严格合炉：按合金大类、温度、厚度组_合炉分组，使用合炉最
大厚度差字段，确保同批内厚度极差≤允许值；单独处理必须单独一炉物料。第七
代（最终版）在第六代基础上，增加炉批间排序优化：同温度组内按厚度降序排列
炉批，不同温度组按温度升序；添加验证表输出相邻炉批的过渡情况。
